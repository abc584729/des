# DES 加密收发链路改造说明

## 1. 项目目标

在保持现有 AD/DA、GT、PAM4 调制及训练同步链路不变的前提下，将原 LSTM 混沌加密替换为 DES 加密，完成发送端加密、接收端解密及端到端数据正确性验证。

本次改造遵循以下原则：

- 复用已经调通的 AD/DA 接口和 GT lane 对齐流程。
- 复用现有训练序列、互相关同步、采样点调整及 PAM4 判决框架。
- 保持现有物理层数据格式不变。
- 分阶段验证，先确认无加密链路，再接入 DES，便于定位问题。

## 2. 技术方案

### 2.1 物理层参数

| 参数 | 设计值 |
| --- | ---: |
| 用户时钟 | 120 MHz |
| 每时钟 PAM4 符号数 | 16 symbols |
| 每符号采样数 | 8 SPS |
| 每时钟采样点数 | 128 samples |
| 单采样点宽度 | 6 bit |
| AD/DA 并行数据宽度 | 768 bit |
| 每时钟逻辑数据量 | 32 bit |

一个120 MHz时钟传输16个 PAM4 符号，每个符号承载2 bit，因此每时钟承载32 bit逻辑数据：

```text
16 symbols × 2 bit/symbol = 32 bit/clock
```

经过8倍采样后，每时钟形成完整的768 bit AD/DA采样帧：

```text
16 symbols × 8 SPS × 6 bit/sample = 768 bit/clock
```

### 2.2 DES 数据适配

DES 每次处理一个64 bit数据块，而现有PAM4链路每时钟承载32 bit，因此一个DES密文块分两个120 MHz时钟发送：

```text
第1拍：cipher[31:0]  → 16个PAM4符号
第2拍：cipher[63:32] → 16个PAM4符号
```

接收端按相同顺序恢复：

```text
第1拍恢复 cipher[31:0]
第2拍恢复 cipher[63:32]
→ 拼接为64 bit密文
→ DES解密
→ 输出64 bit明文
```

### 2.3 发送端数据流

```text
64 bit明文
→ DES加密
→ 64 bit密文
→ 64转2×32 bit
→ 每拍映射为16个PAM4符号
→ 复用原8倍脉冲成形
→ 768 bit DAC数据
→ 原AD/DA及GT发送链路
```

### 2.4 接收端数据流

```text
原AD/DA及GT接收链路
→ 768 bit ADC数据
→ 训练同步、采样点调整及LMS均衡
→ 每拍判决16个PAM4符号
→ PAM4解映射得到32 bit
→ 连续两拍拼接64 bit密文
→ DES解密
→ 明文比较、错误计数
```

## 3. 同步方案

业务同步继续复用现有流程：

```text
发送端循环发送训练序列
→ 接收端检测信号幅度变化
→ 128点互相关确定采样窗口
→ 完成训练序列对齐
→ 发送端在窗口边界切换到有效数据
→ 接收端检测训练数据到有效数据的变化
```

DES模式下，接收端检测到有效数据起点后：

1. 清零32/64 bit拼接状态。
2. 将第一个有效窗口认定为密文低32 bit。
3. 将第二个有效窗口认定为密文高32 bit。
4. 每两个有效窗口组成一个DES密文块。

## 4. 调试计划

### 4.1 阶段一：无加密PAM4链路验证

目的：确认去除混沌加解密后，现有板间物理链路仍能正确传输和恢复PAM4数据。

工作内容：

- 旁路发送端混沌叠加模块。
- 旁路接收端本地LSTM和混沌相减模块。
- 保留原训练同步、8倍脉冲成形、采样点调整、LMS和PAM4判决。
- 发送固定PAM4测试ROM。
- 使用接收端相同参考ROM统计SER。

### 4.2 阶段二：DES端到端链路验证

目的：在阶段一链路稳定的基础上接入DES加解密模块。

工作内容：

- 发送端接入DES加密和64/32 bit适配模块。
- 接收端接入32/64 bit拼接和DES解密模块。
- 使用标准DES测试向量验证算法正确性。
- 使用固定明文ROM完成板间端到端测试。
- 统计密文恢复错误、解密明文错误和连续运行状态。

## 5. 交付内容

### 5.1 验收前

- 收发端测试bitstream。
- 说明文档。

用户依据说明文档完成测试验收；测试结果达到双方约定指标后，确认验收并完成约定款项。

### 5.2 验收后

完成约定款项后，交付：

- 完整发送端Vivado工程。
- 完整接收端Vivado工程。
- 说明文档。
